# Binance Futures Multi-Symbol Auto Bot — GUIDE

이 문서는 최종 완성된 자동매매 코드(`bot-v4.py`)를 기준으로  
**설치 · 설정 · 전략 구조 · 리스크 관리 · 운영 주의사항**을 정리한 가이드입니다.

---

## 1. 봇 개요

- Binance **USDT-M Futures** 전용 다중 심볼 자동매매 봇
- 1시간봉 기준 **시장 국면(Market Regime)** 을 자동 판별
  - `TREND_UP`
  - `TREND_DOWN`
  - `RANGING`
- 국면에 따라 **서로 다른 진입 논리**를 적용하는 듀얼 엔진 구조
- 진입: **시장가**
- 익절(TP): **지정가 트리거 (TAKE_PROFIT)**
- 손절(SL): **시장가 트리거 (STOP_MARKET)**

핵심 목표:

> ❝ 불필요한 구간에서는 쉬고, 유리한 국면에서만 공격한다 ❞

## 2. 설치 방법

### 2.1 Python 환경

- Python 3.10 이상 권장

### 2.2 필수 패키지 설치

```bash
pip install -r requirements.txt
```

### 2.3 환경변수 설정 (.env)

프로젝트 루트에 `.env` 파일을 생성합니다.

```dotenv
BINANCE_API_KEY=YOUR_API_KEY
BINANCE_SECRET_KEY=YOUR_SECRET_KEY
N8N_WEBHOOK_URL=https://n8n-webhook-url (선택)
```

- `N8N_WEBHOOK_URL`은 선택 사항입니다.
- 설정하지 않으면 알림은 비활성화됩니다.

### 2.4 실행

```bash
python bot4.py
```

로그는 콘솔과 `trade.log` 파일에 동시에 기록됩니다.

## 4. 전략 구조

### 4.1 시장 국면 판별 (Market Regime)

**1시간봉 기준**

- ADX(14): 추세 강도
- SMA 기울기: 추세 방향
- 히스테리시스 적용 → 국면 깜빡임 방지

| 국면       | 의미      |
| ---------- | --------- |
| TREND_UP   | 상승 추세 |
| TREND_DOWN | 하락 추세 |
| RANGING    | 횡보장    |

---

### 4.2 국면별 진입 로직

모든 판단은 **확정봉 기준(-2)** 으로 수행됩니다.

#### TREND_UP

- 조건: 저가가 중심선(mid) 터치 또는 이탈
- 진입: **LONG**
- 성격: 눌림목 매매

#### TREND_DOWN

- 조건: 고가가 중심선(mid) 터치 또는 이탈
- 진입: **SHORT**
- 성격: 반등 매도

#### RANGING

- 상단 엔벨로프 진입 → **SHORT**
- 하단 엔벨로프 진입 → **LONG**
- 상·하단 동시 터치 봉은 진입하지 않음
- 성격: 평균회귀(Mean Reversion)

---

### 4.3 TP / SL 구조

| 구분     | 방식          |
| -------- | ------------- |
| 진입     | 시장가        |
| 익절(TP) | 지정가 트리거 |
| 손절(SL) | 시장가 트리거 |

손익비:

- 손절: **-1%**
- 익절: **+1.5%**
- 모두 **진입가 기준**

> 손절은 항상 시장가로 유지하여 포지션 보호를 최우선으로 합니다.

## 5. 주문 안전 장치

### 고아 포지션 방어 (Critical)

- 시장가 진입 후
- TP 또는 SL 주문 등록 실패 시
- **즉시 반대 방향 시장가 청산** 수행

이는 실전 운용에서 가장 중요한 보호 로직입니다.

---

## 6. 실행 흐름 요약

1. 봇 시작
2. 심볼별 레버리지 설정
3. 반복 루프:
   - 선물 잔고 조회
   - 포지션 상태 확인
   - 포지션 종료 시 잔여 주문 정리
   - 국면 판별
   - 국면별 진입 신호 평가
   - 리스크 기반 수량 계산
   - 주문 실행

---

## 7. 운영 시 주의사항

### 7.1 TP 지정가 특성

- 트리거 후 지정가 주문이 생성됨
- 급변 시 TP가 체결되지 않을 수 있음
- SL은 시장가이므로 계좌 보호는 유지됨

권장:

- TP 지정가를 트리거 가격보다 약간 보수적으로 설정

### 7.2 실거래 전 필수 테스트

- 최소 주문 수량
- TP / SL 체결 여부
- 포지션 종료 후 주문 정리 여부

→ 반드시 **소액 또는 페이퍼 테스트** 선행
